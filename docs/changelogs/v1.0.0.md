# Release Notes — v1.0.0

**Date:** 2025
**Type:** Initial release

---

## Summary

First working version of the CWL Performance Tracker. Implemented as an Electron
desktop app that fetches live Clash of Clans Clan War League data and computes
fair bonus distribution rankings using a custom weighted performance scoring algorithm.

---

## Features

### Core App
- Electron desktop app with CoC REST API integration
- API token stored in local `api_config.json` (gitignored)
- 15v15 and 30v30 war format support

### Scoring Algorithm
- Custom weighted performance score per attack
- Accounts for: stars earned, destruction %, attacker vs defender strength (TH level + map position)
- TH strength values manually calibrated for TH1–TH17
- Missed attack penalty: -20 flat score
- `POINTS_BUFFER = 100` to stabilize difficulty ratios
- Best N of M averaging (default: best 6 of 7)

### Player Management
- Per-player attack data editor modal
- War day lineup viewer (clan vs enemy side by side)
- Player TH level selector (custom dropdown with icons)

### Bonus Distribution
- Manual wars won + CWL finished controls
- Manual league tier selection
- Bonus checkboxes in the rankings table (limited to computed bonus count)

### Data Persistence
- Save / Load JSON export via native file dialogs
- Auto-refresh with configurable interval (0 = startup only)
- `localStorage` persistence between sessions (players, war data, settings)

### Settings
- Decimal places for score display
- Best attacks to average (N in best N of M)
- Auto-refresh interval

## Architecture Decisions Made

See `docs/architecture/decisions.md`:
- ADR-001: Use Electron (not a web app)
- ADR-002: Never store computed scores in localStorage
- ADR-003: Data freeze safeguard in processApiData()
- ADR-004: TH strength values are manually calibrated
- ADR-005: API token in api_config.json (not env vars)
- ADR-006: Promise.all for parallel war tag fetching
