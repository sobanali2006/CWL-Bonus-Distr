# Release Notes — v1.2.1

**Date:** 2026-04-01
**Type:** Feature / Infrastructure Update

---

## Summary

Implemented the **Public Server Proxy** infrastructure to bypass dynamic IP restrictions enforced by the Clash of Clans API. Initially deployed on Render.com with a NAT Gateway IP aggregation strategy.

> ⚠️ **Note:** The Render.com deployment was later superseded by Google Cloud Platform
> (see `v1.2.2.md` and ADR-008). This release note is preserved for historical record.

---

## Features

### Proxy Server Implementation
- Created `proxy-server/server.js` — a lightweight Node.js/Express transparent proxy that forwards any CoC API path to `api.clashofclans.com` using a server-side token.
- Added `proxy-server/package.json` with Express dependency.
- Implemented wildcard routing (`/coc-proxy/*`) for seamless CoC API path forwarding.
- Used `.pipe()` streaming for memory-efficient response forwarding.

### Electron App Integration
- Added `proxyUrl` optional field to `api_config.json` schema.
- Added `apiRequestViaProxy()` function in `main.js` for proxy requests.
- Implemented silent fallback logic: if proxy returns 502/500, automatically falls back to local API token.

### Documentation
- Created `docs/deep-dives/public-server-architecture.md` with line-by-line code explanations.
- Added ADR-007 (Render.com) to `docs/architecture/decisions.md` (later superseded by ADR-008).
