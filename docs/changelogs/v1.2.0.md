# Release Notes — v1.2.1

**Date:** 2026-04-01
**Type:** Feature / Infrastructure Update

---

## Summary

Pivoted the **Public Server Proxy** implementation to utilize **Render.com**. This completely eliminates the need for developers/users to provide restrictive cloud credit card information while still fully bypassing dynamic IP restrictions enforced by the Clash of Clans API.

---

## Features

### Render.com Proxy Integration
- The Node.js proxy server deployed under `proxy-server/` is completely compatible with Render.com's Free Web Service container architecture.
- Instead of using a single Dedicated Virtual Machine IP, the proxy aggregates Render's strict pool of regional NAT Gateways (e.g. 3 IPs in US-West). By uploading all 3 IPs concurrently into the CoC Developer token, it forces a flawless connectivity bypass.
- Endured 45-second "Cold-Starts" are mitigated via silent, robust Electron fallback architecture.

### Documentation Updates
- Re-composed the `proxy-server/README.md` step-by-step documentation explicitly for GitHub-native Render deployments.
- Substituted `ADR-007` to reflect the pivot from Oracle virtualization to Render PaaS. 
- Overhauled User-Architecture deep dives resolving strict financial bottlenecks.
